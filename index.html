Planning a simple website

Information Architechture is important for planning and launching a website

  1. Note down what you want to have common to every page.
  2. Draw a rough sketch of what you might want the structure of each page to look like.
    What each block is going to be.
  3. Brainstorm the content you want on your website on paper.
  4. Sort the brainstormed ideas into groups.
  5. Now sketch a rough sitemap, with a bubble for each page.
    The home page will be in the center with lines connected to the other pages.
    Include possible notes on how things might be presented.

Debugging html

HTML is considered a permissive language. This means that the output is guaranteed regardless of syntax errors.
The browser will markup the errors to its best ability and display the page.

  • Syntax errors: Spelling errors in code.
  • Logic errors: Error within logic where the program runs incorrectly.

HTML validation

To make sure your HTML or CSS is well-formed, run it through the Markup Validation Service

This service goes through your markup and reports any faulty code.

Audio and Video

Use <video> with src attribute

Or use <source>with src attribute nested in video tag

The poster attribute allows custom thumbnail for videos.

The <audio> element works similar to the video except width, height and poster attributes are unsupported.

Video text tracks

WebVTT is a format for displaying timed <track> for <video> elements.

set the kind attribute to either subtitles, captions, or descriptions

Embedding

iframes allow you to imbed other documents into your current webpage.

<iframe width="250" height="15%" src="https://www.youtube.com/embed/oEZAJls3Wmc" frameborder="0" allow="accelerometer; autoplay; encrypted-media; gyroscope; picture-in-picture" allowfullscreen></iframe>

embed and object elements are used to embed external content like java applets, flash, and svg. Somewhat outdated.

Security

  • Use sandbox attribute to wrap embedded content for security purposes, to prevent attack vectors.
  • Only embed when necessary.
  • Use HTTPS as it is the encrypted version of HTTP.
  • Use CSP directives to configure your website and iframes from being embedded on other sites.

Vector graphics

SVG is great for scalable images that give a crips look, unlike raster graphics.

CSS and javascript can also edit SVG

      <svg version="1.1"
        baseProfile="full"
        width="250" height="15%"
        xmlns="http://www.w3.org/2000/svg">
        <rect width="100%" height="100%" fill="black" />
        <circle cx="125" cy="70" r="60" fill="green" />
        <text x="70" y="70" font-size="30" fill="white" stroke="black">SVG text</text>
      </svg>

You can also use external svg files, and link them using the <img> tag, but with this method you cannot edit the element with CSS or javascript.

Responsive Images

In the <img> tag use srcset and sizes attribute to include additional responsive sizes for different screens.

      <img srcset="banskel-320w.jpg 320w,
                   banskel-480w.jpg 480w,
                   banskel-800w.jpg 800w"
           sizes="(max-width: 320px) 280px,
                  (max-width: 480px) 440px,
                  800px"
           src="banskel-800w.jpg" alt="img description">

You can omit the sizes attribute and instead edit the width: of the img within CSS.

Use the <picture> element to wrap a <source> tag with media and srcset attribute to serve cropped images for differnet layouts.

<picture>
  <source media="(max-width: 799px)" srcset="banskel-480w-close-portrait.jpg">
  <source media="(min-width: 800px)" srcset="banskel-800w.jpg">
  <img src="banskel-800w.jpg" alt="banskel standing up holding his daughter tomatoisland">
</picture>

Forms

Common form elements include form, label, input, textarea, and button

Single line text fields remove line breaks before sending data.

Email address fields can have the multiple attribute to allow multiple email addresses seperated by commas.

Password fields replace text with asterics. Secure connection (https) is recommended when sending password data.

Search fields saves autocomplete information across pages on the same website.
